Git vs Github

Git

Git is a specialized computer program that acts like a highly advanced, collaborative time machine for files. It automatically records the history of changes made to a project, allowing creators to save different versions of their work, safely experiment with new ideas on the side, and roll back to any exact moment in the past if something goes wrong.

Github

GitHub is an online, cloud-based platform that hosts Git projects. While Git is a tool you run locally on your own computer, GitHub is the website where you upload those project histories so other people can see them, download them, and contribute to them.

The Daily Loop

  1. Edit (Working Directory): You modify, create, or delete files in your project. At this point, the changes only exist as live, un-saved edits on your local machine.
  2. Stage (Staging Area): run git add to move your edited files into the staging area. You choose what you want to keep in your next save point.
  3. Commit (Local Repository): git commit -m "your message". This takes a permanent snapshot of your project's history on your computer.
  4. Push (Remote Repository): Uploads your commits over the internet to a remote server like github.

Git Commands

  • git init : Creates a brand new, empty local Git repository.
  • git clone : Copies an existing repository onto your local machine.
  • git status : Shows the current state of your working directory and staging area.
  • git add : Stages your file changes, preparing them to be saved.
  • git commit : command saves a permanent snapshot of your staged changes into your local Git repository's history.
  • git branch : Lists, creates, or deletes safe isolated environments for developing features.
  • git pull : Fetches and merges updates from the remote server into your local workspace.
  • git push : uploads your local repository commits to a remote repository (usually github).

Things I Should Not Commit

  • Sensitive Information : API keys, tokens, passwords & credectials, general personal data
  • Local Environment & OS Metadata: .env, .vscode, .idea, .DS_store, *.log
  • Files Created From the source code.
